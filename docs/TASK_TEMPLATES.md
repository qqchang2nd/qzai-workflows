# QZAI Task Templates

Task templates define what each agent is asked to do when a command is dispatched.
They are generated by `hooks/slash-bridge-v1/src/task/*.js` and sent via A2A to the target agent.

---

## Template Structure

Every task template has this shape:

```js
{
  type: string,          // task type identifier
  context: {             // request context (data)
    repo, prNumber, headSha, baseSha, round, ...
  },
  instructions: string,  // natural language instructions for the agent
  writeback: {           // how to write results back to GitHub
    type: 'update-comment',
    issueNumber,
    checkRun?: { name, headSha, conclusion? }
  }
}
```

## `plan` — Plan Task

**File**: `src/task/plan.js`
**Default Agent**: lixunhuan
**Type**: `plan`

**Context**:
- `repo`, `issueNumber`, `issueBody`, `requestedBy`

**Instructions summary**:
1. Read Issue #N description
2. Generate PLAN.md with: Goals / Non-Goals / Approach / Risks / Rollback / DoD
3. Push to branch `qzai/plan/issue-N`
4. Create Plan PR (label: `qzai:plan`)
5. Comment Plan PR link on Issue

## `review` — Code Review Task

**File**: `src/task/review.js`
**Default Agent**: lixunhuan
**Type**: `code-review`

**Context**:
- `repo`, `prNumber`, `headSha`, `baseSha`, `round`, `isIncremental`

**Instructions summary**:
- Round 1: Review full PR diff
- Round 2+: Review only new commits since last review (`isIncremental: true`)
- Output format: `[severity: critical|high|medium|low] file:line - description - fix suggestion`
- Conclusion: Pass ✅ or N blocking issues ❌
- Submit via GitHub Review API: APPROVE (pass) or REQUEST_CHANGES (blocking)

## `followup` — Followup Task (two modes)

**File**: `src/task/followup.js`
**Default Agent**: lengyan

### Mode: `notify` (auto-triggered after each review with blocking issues)

**Type**: `followup-notify`

**Context**:
- `prAuthor`, `round`, `blockingIssues[]`

**Instructions summary**:
- Comment on PR @prAuthor
- List blocking issues from current round
- Request: push fixes, system will auto-trigger Round N+1

### Mode: `escalation` (auto-triggered after max_rounds reached)

**Type**: `followup-escalation`

**Context**:
- `prAuthor`, `originalReviewer`, `allUnresolvedIssues[]`, `maxRounds`

**Instructions summary**:
- Comment on PR @prAuthor and @originalReviewer
- Summarize all unresolved issues across all rounds
- Request human decision: fix / accept with justification / close PR
- Sets check-run: `failure` (`MAX_ROUNDS_REACHED`)

## Review Loop State Machine

```
startReviewRound()
  → round=1, status=reviewing

completeReviewRound(blockingIssues=[])
  → status=approved, needsFollowup=false

completeReviewRound(blockingIssues=[...])
  if round < maxRounds:
    → status=pending_fix, followupMode='notify'
  if round >= maxRounds:
    → status=max_reached, followupMode='escalation'

handlePrPush() [on PR synchronize]
  if status=pending_fix:
    → creates round=N+1, status=reviewing
  if status=max_reached:
    → no-op (silent)
```

## Configuration

| `QZAI_A2A_TOKEN` | (none) | Bearer token for A2A endpoint |
